# Plan: Build FLM Bridge for Correct NPU Inference

## Problem

Our standalone npu-infer fails because:
1. We CPU-dequantize Q4NX weights (wrong format)
2. We submit raw opcode=3 without instructions (produces identity)

## Solution

Use FLM's instruction generators to produce correct NPU runlists per layer.
Load FLM symbols via dlopen, call `gen_layer_seq` to generate instructions,
and submit them via XRT with opcode=3.

## Architecture: Use gen_layer_seq

`gen_layer_seq` generates ALL instructions for one transformer layer in one call.
This is simpler than calling each sub-function individually.

The plan:
1. Create `FlmBridge` singleton that loads all FLM libs and provides clean C++ methods
2. For each layer: call `gen_layer_seq(seq, layer_idx)`, extract instructions
3. For lm_head: call `gen_lm_head_seq(seq)`, extract instructions  
4. Pre-generate all layer instruction buffers at init time
5. During inference, submit pre-generated instructions with correct activation BOs

### Weight BO Allocation

For `gen_layer_seq` to work, weight BOs must be allocated and registered. The FLM
sequence object needs to know about weight BOs via `npu_xclbin_manager`. But
`gen_layer_seq` generates DMA instructions — it doesn't access BOs directly.

The key: the instructions reference BOs by their device addresses. We need the
weight BOs at the correct device addresses. FLM allocates weight BOs internally
via `npu_xclbin_manager` when loading weights.

Alternative simpler approach: Just use `qwen3_npu_sequence` functions to generate
the instruction sequence, then our engine submits the instructions. The weight
BOs are referenced by their DDR addresses in the instructions.

Actually, looking at `full_pipeline.cpp` more carefully:
- The weight BOs are passed to xclbin kernel as arguments (bo_a, bo_b, bo_c)
- The instructions generated by `gen_layer_seq` reference internal weight buffers
- We'd need the weight data in the format the xclbin expects

**Simplest working path:** Use the `full_pipeline.cpp` approach but for 1 token,
calling `gen_layer_seq` for each of 28 layers + `gen_lm_head_seq`. Generate all
instructions upfront, then submit sequentially.
